# MySQL 数据库的创建与删除操作

在 MySQL 中，数据库（Database）是存储表和数据的容器。以下是创建和删除数据库的详细操作指南：


## 1. 创建数据库（CREATE DATABASE）

### 基本语法
```sql
CREATE DATABASE [IF NOT EXISTS] 数据库名 [CHARACTER SET 字符集] [COLLATE 排序规则];
```

- **`IF NOT EXISTS`**：可选，避免创建已存在的数据库时报错。
- **`CHARACTER SET`**：可选，指定数据库默认字符集（如 `utf8mb4` 支持 emoji）。
- **`COLLATE`**：可选，指定字符集的排序规则（如 `utf8mb4_general_ci` 不区分大小写）。


### 示例
1. 创建一个名为 `mydb` 的数据库：
   ```sql
   CREATE DATABASE mydb;
   ```

2. 安全创建（避免重复）并指定字符集：
   ```sql
   CREATE DATABASE IF NOT EXISTS mydb 
   CHARACTER SET utf8mb4 
   COLLATE utf8mb4_general_ci;
   ```


## 2. 查看数据库列表

创建后可通过以下命令确认：
```sql
-- 查看所有数据库
SHOW DATABASES;

-- 查看指定数据库的创建信息（含字符集）
SHOW CREATE DATABASE mydb;
```


## 3. 删除数据库（DROP DATABASE）

### 基本语法
```sql
DROP DATABASE [IF EXISTS] 数据库名;
```

- **`IF EXISTS`**：可选，避免删除不存在的数据库时报错。
- **注意**：删除数据库会同时删除其中所有表和数据，操作不可逆！


### 示例
删除名为 `mydb` 的数据库：
```sql
-- 谨慎操作！
DROP DATABASE IF EXISTS mydb;
```


## 4. 切换当前数据库

创建数据库后，需切换到该数据库才能操作其中的表：
```sql
USE mydb;
```

执行后，后续的表操作（创建表、插入数据等）将默认在 `mydb` 中进行。


## 权限说明
- 创建/删除数据库需要 `CREATE` 和 `DROP` 权限，通常只有管理员（如 `root` 用户）拥有。
- 普通用户需管理员授权后才能执行这些操作。

操作时请务必确认数据库名称，避免误删重要数据！


## 示例展示
```sql
CREATE DATABASE test DEFAULT CHARACTER SET utf8;  -- 创建test数据库

SHOW DATABASES; -- 查看数据库

DROP DATABASE test; -- 删除数据库

USE test; -- 选择数据库
```
