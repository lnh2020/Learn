# Redis 简介
Redis 是一款开源的**高性能键值对（Key-Value）数据库**，全称为 Remote Dictionary Server，以内存存储为核心，兼具持久化能力，广泛用于缓存、会话存储、实时数据处理等场景。


## 1. 核心特性
- **基于内存存储**：数据主要存于内存中，读写速度极快，单机 QPS 可轻松达到数万级别。
- **支持多数据结构**：不仅能存储字符串，还支持哈希（Hash）、列表（List）、集合（Set）、有序集合（Sorted Set）等多种结构，满足复杂业务需求。
- **持久化机制**：提供 RDB（快照）和 AOF（日志）两种持久化方式，可将内存数据定期或实时写入磁盘，避免数据丢失。
- **高可用与分布式**：支持主从复制、哨兵（Sentinel）模式和集群（Cluster）模式，能实现故障自动切换和水平扩容。
- **丰富附加功能**：内置发布订阅（Pub/Sub）、事务、过期键自动删除、Lua 脚本等功能，扩展性强。


---

## 2. 典型应用场景
- **缓存加速**：作为数据库的“前置缓存”，存储高频访问数据（如商品详情、用户信息），减少数据库查询压力，提升系统响应速度。
- **会话存储**：在分布式系统中，将用户登录会话（Session）数据存入 Redis，实现多服务间的会话共享。
- **实时排行榜**：利用有序集合（Sorted Set）的排序特性，快速实现积分排行、销量排行等实时榜单功能。
- **计数器与限流**：通过原子操作实现点赞数、阅读量等计数器，或基于 Redis 实现接口限流（如令牌桶、漏桶算法）。
- **消息队列**：借助列表（List）的 `LPUSH`/`RPOP` 命令，实现简单的消息生产与消费，作为轻量级消息队列使用。


---

## 3. 与 MySQL 的核心差异
| 对比维度       | Redis                  | MySQL                  |
|----------------|------------------------|------------------------|
| **数据模型**   | 键值对（支持多结构）   | 关系型（表结构）       |
| **存储介质**   | 主要在内存（可持久化） | 主要在磁盘             |
| **读写性能**   | 极高（微秒级响应）     | 中等（毫秒级响应）     |
| **适用数据**   | 高频访问、临时数据     | 结构化、持久化数据     |
| **事务支持**   | 弱事务（部分原子操作） | 强事务（ACID 兼容）    |


---

## 4. 基础操作示例
1. **启动 Redis 服务**：`redis-server`（默认端口 6379）。
2. **连接 Redis 客户端**：`redis-cli -h 主机地址 -p 端口`（本地默认直接输入 `redis-cli`）。
3. **数据操作（以字符串为例）**：
   - 插入数据：`SET username "zhangsan"`
   - 查询数据：`GET username`（返回 "zhangsan"）
   - 设置过期时间：`SET key value EX 30`（30 秒后键自动过期）
4. **查看所有键**：`KEYS *`
