在 Linux 命令行中，通配符（Wildcards）用于匹配文件名或路径，简化文件批量操作。常用的通配符及其用法如下：

## 1. `*`（星号）
+ **功能**：匹配任意数量的任意字符（包括零个字符）
+ **示例**：
```bash
# 列出当前目录下所有 .txt 文件
ls *.txt

# 列出所有以 "log" 开头的文件
ls log*

# 复制所有以 "data_" 开头且以 ".csv" 结尾的文件到 backup 目录
cp data_*.csv backup/
```

## 2. `?`（问号）
+ **功能**：匹配**单个**任意字符
+ **示例**：
```bash
# 匹配文件名长度为3个字符的文件（如 abc、123 等）
ls ???

# 匹配以 "file_" 开头，后面跟1个任意字符，再以 ".txt" 结尾的文件
# 如 file_a.txt、file_1.txt 等
ls file_?.txt
```

## 3. `[]`（方括号）
+ **功能**：匹配括号内的**任意一个字符**（字符集）
+ **用法**：
    - `[abc]`：匹配 a、b、c 中的任意一个
    - `[0-9]`：匹配 0-9 中的任意一个数字
    - `[a-z]`：匹配小写字母 a-z 中的任意一个
    - `[^abc]` 或 `[!abc]`：匹配**不在**括号内的任意一个字符（取反）
+ **示例**：
```bash
# 列出所有以 a、b、c 开头的 .sh 文件
ls [abc]*.sh

# 列出文件名包含数字的文件
ls *[0-9]*

# 列出不以 .log 结尾的文件（取反）
ls *[!.]log
```

## 4. `{}`（花括号）
+ **功能**：生成**多个字符串组合**（不是匹配已有文件，而是生成新的模式）
+ **示例**：
```bash
# 创建多个文件（file1.txt、file2.txt、file3.txt）
touch file{1,2,3}.txt

# 复制 file.txt 到 dir1、dir2、dir3 目录
cp file.txt {dir1,dir2,dir3}/

# 生成连续序列（如 1-5 或 a-c）
mkdir dir{1..5}       # 创建 dir1 到 dir5 目录
touch {a..d}.md       # 创建 a.md 到 d.md 文件
```

## 5. 通配符组合使用
通配符可以组合使用，实现更复杂的匹配：

```bash
# 匹配以 "img_" 开头，中间是2个数字，结尾是 .jpg 或 .png 的文件
ls img_[0-9][0-9].{jpg,png}

# 复制所有 .txt 和 .pdf 文件到 archive 目录
cp *.{txt,pdf} archive/
```

## 注意事项
+ 通配符由 Shell（如 bash）解析，而非命令本身，因此适用于所有接受文件名作为参数的命令
+ `*` 和 `?` 是 "匹配已有文件"，`{}` 是 "生成模式"（即使文件不存在也会生成对应的字符串）
+ 某些命令（如 `grep`）中使用的 `*` 或 `?` 属于正则表达式，与 Shell 通配符含义不同，需注意区分

通配符是提高命令行效率的重要工具，熟练使用可以大幅简化批量文件操作。
