在 Linux 系统中，查看文件内容的命令有很多，适用于不同场景（如文件大小、查看方式、实时监控等）。以下是常用的文件内容查看命令：

## 1. `cat` - 一次性显示整个文件内容
适合查看**小型文件**，直接输出全部内容到终端。

```bash
cat 文件名          # 显示文件全部内容
cat -n 文件名       # 显示内容并显示行号
cat -b 文件名       # 显示内容并显示行号（忽略空行）
cat file1 file2     # 同时显示多个文件内容
```

示例：

```bash
cat /etc/hosts      # 查看主机配置文件
cat -n script.sh    # 显示脚本内容并带行号
```

## 2. `more` - 分页查看文件（向前翻页受限）
适合查看**较长文件**，支持按页浏览（空格键翻页，`Enter` 键逐行滚动）。

```bash
more 文件名
```

操作键：

+ 空格键：显示下一页
+ `Enter`：显示下一行
+ `b`：回退上一页（部分版本支持）
+ `q`：退出查看

示例：

```bash
more /var/log/messages  # 分页查看系统日志
```

## 3. `less` - 更强大的分页查看工具
`more` 的增强版，支持向前 / 向后翻页、搜索等功能，适合查看**大文件**。

```bash
less 文件名
```

常用操作：

+ 上下方向键：逐行滚动
+ `PageUp`/`PageDown`：翻页
+ `/关键词`：向下搜索关键词（按 `n` 找下一个，`N` 找上一个）
+ `?关键词`：向上搜索关键词
+ `q`：退出

示例：

```bash
less /etc/passwd  # 查看用户信息文件，支持搜索用户名
```

## 4. `head` - 查看文件开头内容
默认显示文件前 10 行，可指定行数。

```bash
head 文件名             # 显示前10行
head -n 行数 文件名     # 显示前N行
head -c 字节数 文件名   # 显示前N个字节
```

示例：

```bash
head -n 5 /etc/profile  # 查看环境变量配置文件的前5行
head -c 100 data.bin    # 查看二进制文件的前100个字节
```

## 5. `tail` - 查看文件末尾内容
默认显示文件后 10 行，常用于查看日志文件，支持实时监控新增内容。

```bash
tail 文件名             # 显示后10行
tail -n 行数 文件名     # 显示后N行
tail -f 文件名          # 实时跟踪文件新增内容（按Ctrl+C退出）
tail -F 文件名          # 与-f类似，但文件被删除重建后仍能继续跟踪
```

示例：

```bash
tail -n 20 /var/log/nginx/access.log  # 查看nginx访问日志最后20行
tail -f /var/log/syslog               # 实时监控系统日志新增内容
```

## 6. `nl` - 显示内容并添加行号
与 `cat -n` 类似，但行号格式更灵活（默认忽略空行）。

```bash
nl 文件名              # 显示内容并添加行号（忽略空行）
nl -ba 文件名          # 显示内容并为所有行（包括空行）添加行号
```

示例：

```bash
nl script.py  # 查看Python脚本并显示行号
```

## 7. `tac` - 反向显示文件内容
与 `cat` 相反，从最后一行开始显示到第一行。

```bash
tac 文件名
```

示例：

```bash
tac /var/log/dmesg  # 反向查看系统启动日志（最新内容在前）
```

## 8. `od` - 查看二进制文件
以**八进制、十六进制**等格式查看二进制文件（如可执行文件、图片等）。

```bash
od -t x1 文件名  # 以十六进制单个字节显示
od -t d2 文件名  # 以十进制双字节显示
```

示例：

```bash
od -t x1 /bin/ls  # 以十六进制查看ls命令的二进制内容
```

## 选择建议：
+ 小文件：直接用 `cat`
+ 大文件分页浏览：用 `less`（功能最全）
+ 只看开头 / 末尾：用 `head` 或 `tail`
+ 实时监控日志：用 `tail -f`
+ 二进制文件：用 `od`

这些命令是日常查看文件内容的基础，结合管道（`|`）还能实现更复杂的操作，例如 `cat largefile.txt | grep "error"` 筛选包含 "error" 的行。
