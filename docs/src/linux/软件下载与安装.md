#### 配置yum源
<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">在 CentOS、RHEL 等基于 RPM 的 Linux 发行版中，</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">yum</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">（或新一代的</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">dnf</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">）依赖于</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">yum 源</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">（软件仓库）获取软件包。配置合适的 yum 源可以加快下载速度、获取更多软件。以下是配置 yum 源的详细步骤：</font>

##### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">一、yum 源的基本概念</font>
+ **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">yum 源</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：存储软件包的服务器，通过配置文件（</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">.repo</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">）告诉系统仓库地址。</font>
+ **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">配置文件位置</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">/etc/yum.repos.d/</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">目录下，所有</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">.repo</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">结尾的文件都会被 yum 识别。</font>
+ **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">常用源类型</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">官方源：系统默认配置，稳定性高但可能速度慢。</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">第三方源：如 EPEL（提供额外开源软件）、阿里云、网易等镜像源（速度快，适合国内用户）。</font>

##### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">二、配置国内镜像源（以阿里云为例）</font>
<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">国内镜像源（如阿里云、网易）通常比官方源速度更快，推荐优先配置。</font>

###### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">1. 备份默认源（可选但推荐）</font>
**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
# 进入源配置目录
cd /etc/yum.repos.d/

# 备份所有默认 repo 文件
sudo mkdir backup
sudo mv *.repo backup/
```

###### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">2. 下载国内镜像源配置文件</font>
<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">以 CentOS 7 为例，下载阿里云的 repo 文件：</font>

**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash

sudo wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

# CentOS 8（注意：CentOS 8 官方已停止维护，需用 Stream 版本的源）

```

<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">其他发行版 / 版本可参考阿里云镜像站：</font>[<font style="color:rgb(9, 105, 218);background-color:rgb(252, 252, 252);">https://developer.aliyun.com/mirror/</font>](https://developer.aliyun.com/mirror/)

##### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">三、添加 EPEL 源（扩展软件仓库）</font>
<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">EPEL（Extra Packages for Enterprise Linux）提供了官方源中没有的大量开源软件，建议安装。</font>

###### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">1. 安装 EPEL 源</font>

##### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">四、手动配置自定义 yum 源（高级操作）</font>
<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">若需要添加特定软件的仓库（如 Docker、Nginx 官方源），可手动创建</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">.repo</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">配置文件。</font>

###### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">示例：添加 Docker 官方源</font>
1. <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">创建 repo 文件：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
sudo vim /etc/yum.repos.d/docker-ce.repo
```

2. <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">写入以下内容（根据官方文档获取最新地址）：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">ini</font>**

```plain
[docker-ce-stable]
name=Docker CE Stable - $basearch
baseurl=https://download.docker.com/linux/centos/$releasever/$basearch/stable
enabled=1
gpgcheck=1
gpgkey=https://download.docker.com/linux/centos/gpg
```

    - `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">[docker-ce-stable]</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：仓库唯一标识（不能重复）</font>
    - `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">name</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：仓库名称（自定义）</font>
    - `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">baseurl</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：仓库地址（必须正确）</font>
    - `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">enabled=1</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：启用该仓库（0 为禁用）</font>
    - `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">gpgcheck=1</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：验证软件包签名（安全推荐开启）</font>

##### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">五、验证与更新 yum 源</font>
<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">配置完成后，需更新 yum 缓存使配置生效：</font>

###### **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">清理旧缓存</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**


###### **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">生成新缓存</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

##### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">六、常见问题解决</font>
1. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">“Could not retrieve mirrorlist” 错误</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">检查网络连接（</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">ping mirrors.aliyun.com</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">）</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">确认 repo 文件中的</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">baseurl</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">是否正确（尤其是版本号）</font>
2. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">GPG 验证失败</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">临时禁用 GPG 检查：</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo yum install 软件名 --nogpgcheck</font>`
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">或添加正确的</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">gpgkey</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">地址到 repo 文件</font>
3. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">源冲突</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">禁用不需要的仓库：</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo yum-config-manager --disable 仓库名</font>`
    - <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">或直接删除对应的</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">.repo</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">文件</font>

<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">通过配置合适的 yum 源，能显著提升软件安装速度和可用性，国内用户优先推荐阿里云、网易等镜像源。</font>

| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">卸载软件（保留配置）</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo apt remove 软件名</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">彻底卸载（删除配置）</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo apt purge 软件名</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">升级指定软件</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo apt install --only-upgrade 软件名</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">升级所有软件</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo apt upgrade</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">搜索软件</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">apt search 关键词</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">查看软件信息</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">apt show 软件名</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">列出已安装软件</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">apt list --installed</font>` |
| **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">清理无用依赖</font>** | `<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo apt autoremove</font>` |


**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">示例</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>

**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
# 更新缓存并安装 MySQL
sudo apt update && sudo apt install mysql-server

# 升级系统所有软件
sudo apt upgrade

# 彻底卸载 LibreOffice
sudo apt purge libreoffice*
```

### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">三、通用补充命令（适用于多数发行版）</font>
1. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">源码编译安装</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">（无预编译包时使用）：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
# 下载源码包
wget https://example.com/software.tar.gz

# 解压
tar -zxvf software.tar.gz && cd software

# 配置、编译、安装
./configure --prefix=/usr/local/software  # 自定义安装路径
make
sudo make install
```

2. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">二进制包安装</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">（预编译包，无需编译）：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
# 下载二进制包（以 Node.js 为例）
wget https://nodejs.org/dist/v20.9.0/node-v20.9.0-linux-x64.tar.xz

# 解压并移动到系统目录
tar -Jxvf node-v20.9.0-linux-x64.tar.xz
sudo mv node-v20.9.0-linux-x64 /usr/local/nodejs

# 添加到环境变量（方便直接运行）
echo 'export PATH=/usr/local/nodejs/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

3. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">查看软件安装路径</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
which 软件名    # 查看可执行文件路径（如 which python3）
whereis 软件名  # 查看软件相关文件路径（如 whereis nginx）
```

4. **<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">验证软件是否安装</font>**<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">：</font>**<font style="color:rgb(249, 250, 251);background-color:rgb(252, 252, 252);">bash</font>**

```bash
dpkg -l 软件名  # Debian 系（如 dpkg -l nginx）
rpm -q 软件名   # RedHat 系（如 rpm -q nginx）
```

### <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">四、注意事项</font>
1. <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">执行安装 / 卸载 / 升级操作通常需要管理员权限（</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">sudo</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">或</font><font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">root</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">）。</font>
2. <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">优先使用官方仓库安装软件，安全性和兼容性更有保障。</font>
3. <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">升级系统前建议备份重要数据，避免版本冲突导致问题。</font>
4. <font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">第三方软件（源码 / 二进制包）需手动管理升级和卸载，建议记录安装路径。</font>

<font style="color:rgb(0, 0, 0);background-color:rgb(252, 252, 252);">根据使用的 Linux 发行版选择对应命令，熟练掌握包管理工具能高效完成软件全生命周期管理。</font>
