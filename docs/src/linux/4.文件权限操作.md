## 1. 使用 chmod命令修改文件权限
`chmod` 命令用于修改文件 / 目录的权限，格式如下：



```bash
chmod [选项] 权限设置 文件名/目录名
```

### 1. 符号模式修改权限
通过符号指定**用户范围**、**操作符**和**权限**：

+ 用户范围：`u`（所有者）、`g`（所属组）、`o`（其他用户）、`a`（所有用户，默认）
+ 操作符：`+`（添加权限）、`-`（移除权限）、`=`（设置权限，覆盖原有）
+ 权限：`r`、`w`、`x`

示例：



```bash
# 给文件所有者添加执行权限
chmod u+x file.txt

# 移除所属组的写权限
chmod g-w file.txt

# 给所有用户设置读权限（覆盖原有）
chmod a=r file.txt

# 同时修改多组权限：所有者rwx，所属组rx，其他用户无权限
chmod u=rwx,g=rx,o=- file.txt
```

### 2. 数字模式修改权限
每个权限对应一个数字，通过累加计算每组权限的数值：

+ `r` = 4
+ `w` = 2
+ `x` = 1
+ 无权限 = 0

每组权限的数值范围是 0~7，三组权限组合成一个 3 位数字（如 `754`）：

+ 第一位：所有者权限（`7` = `4+2+1` = `rwx`）
+ 第二位：所属组权限（`5` = `4+1` = `r-x`）
+ 第三位：其他用户权限（`4` = `4` = `r--`）

示例：`rwxr-xr--` 对应数字表示为 `754`。

直接用 3 位数字设置权限（更简洁）：



```bash
# 设置权限为 rwxr-xr--（754）
chmod 754 file.txt

# 设置目录权限为 rwxr-x---（750）
chmod 750 dir/

# 给所有用户添加读权限（原权限基础上，相当于 a+r）
# 例如原权限为 640（rw-r-----），执行后为 644（rw-r--r--）
chmod +r file.txt
```

:::info
在 Linux 系统中，修改文件或目录的所属用户组可以使用 `chgrp` 命令，也可以通过 `chown` 命令同时修改所有者和所属组。以下是具体操作方法：
:::

## 2. 使用 `chgrp` 命令（专门修改所属组）
`chgrp` 是专门用于修改文件 / 目录所属用户组的命令，基本格式：



```bash
chgrp [选项] 新组名 文件名/目录名
```

**常用选项**：

+ `-R`：递归修改目录及其子目录中所有文件的所属组

**示例**：



```bash
# 将 file.txt 的所属组改为 developers
chgrp developers file.txt

# 递归修改 /data 目录及其所有子内容的所属组为 project
chgrp -R project /data
```

## 3. 使用 `chown` 命令（同时修改所有者和所属组）
`chown` 主要用于修改文件的所有者，但也可以通过 `所有者:所属组` 的格式同时修改所属组，基本格式：



```bash
chown [选项] 所有者:新组名 文件名/目录名
```

**说明**：

+ 若只需要修改所属组，可以将所有者部分留空（但冒号不能省略），如 `:新组名`

**示例**：



```bash
# 只修改 file.txt 的所属组为 testers（保持所有者不变）
chown :testers file.txt

# 同时修改所有者为 alice 和所属组为 design
chown alice:design image.png

# 递归修改 /docs 目录的所属组为 editors（保持所有者不变）
chown -R :editors /docs
```

## 注意事项
1. 执行这些命令需要管理员权限（`root` 或通过 `sudo`），普通用户只能修改自己拥有的文件的所属组，且新组必须是该用户所属的组。
2. 修改目录的所属组时，若需要子目录和文件同步变更，务必加上 `-R` 选项（递归操作）。
3. 可以通过 `ls -l` 命令验证修改结果，查看文件信息中第四个字段（所属组）是否已更新。

例如，修改后执行 `ls -l file.txt`，输出中第四个字段应为新设置的用户组名称。
