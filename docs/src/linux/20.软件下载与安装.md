在 Linux 系统中，软件的下载、安装、删除和升级等管理操作主要通过包管理工具完成，不同发行版使用的工具略有差异。以下是基于主流发行版的常用软件管理命令：

## 一、RedHat 系（CentOS、RHEL、Fedora）
主要使用 `yum`（CentOS 7 及以下）和 `dnf`（CentOS 8+、Fedora），两者用法类似，`dnf` 是 `yum` 的升级版本。

| **操作** | `**yum**`<br>**命令** | `**dnf**`<br>**命令** |
| :--- | :--- | :--- |
| **更新软件源缓存** | `sudo yum makecache` | `sudo dnf makecache` |
| **安装软件** | `sudo yum install 软件名` | `sudo dnf install 软件名` |
| **卸载软件** | `sudo yum remove 软件名` | `sudo dnf remove 软件名` |
| **升级指定软件** | `sudo yum update 软件名` | `sudo dnf update 软件名` |
| **升级所有软件** | `sudo yum update` | `sudo dnf upgrade` |
| **搜索软件** | `yum search 关键词` | `dnf search 关键词` |
| **查看软件信息** | `yum info 软件名` | `dnf info 软件名` |
| **列出已安装软件** | `yum list installed` | `dnf list installed` |
| **清理缓存** | `sudo yum clean all` | `sudo dnf clean all` |


**示例**：

```bash
# 安装 Nginx
sudo dnf install nginx

# 升级 Firefox
sudo dnf update firefox

# 卸载 Git
sudo dnf remove git
```

```bash
# 安装 rpm 包，显示详细过程并展示进度条
sudo rpm -ivh package.rpm

# 安装 deb 包，显示详细过程
sudo dpkg -iv package.deb

# 复制目录，显示详细过程
cp -rv /source/dir /target/dir
```

`-i` 核心是 “安装”

`-v` 是 “详细输出”

`-h` 多为 “人类可读格式” 或 “帮助”


### 配置yum源
在 CentOS、RHEL 等基于 RPM 的 Linux 发行版中，`yum`（或新一代的 `dnf`）依赖于 **yum 源**（软件仓库）获取软件包。配置合适的 yum 源可以加快下载速度、获取更多软件。以下是配置 yum 源的详细步骤：

#### 一、yum 源的基本概念
+ **yum 源**：存储软件包的服务器，通过配置文件（`.repo`）告诉系统仓库地址。
+ **配置文件位置**：`/etc/yum.repos.d/` 目录下，所有 `.repo` 结尾的文件都会被 yum 识别。
+ **常用源类型**：
    - 官方源：系统默认配置，稳定性高但可能速度慢。
    - 第三方源：如 EPEL（提供额外开源软件）、阿里云、网易等镜像源（速度快，适合国内用户）。

#### 二、配置国内镜像源（以阿里云为例）
国内镜像源（如阿里云、网易）通常比官方源速度更快，推荐优先配置。

#### 1. 备份默认源（可选但推荐）
```bash
# 进入源配置目录
cd /etc/yum.repos.d/

# 备份所有默认 repo 文件
sudo mkdir backup
sudo mv *.repo backup/
```

#### 2. 下载国内镜像源配置文件
以 CentOS 7 为例，下载阿里云的 repo 文件：

```bash
# CentOS 7
sudo wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-7.repo

# CentOS 8（注意：CentOS 8 官方已停止维护，需用 Stream 版本的源）
sudo wget -O /etc/yum.repos.d/CentOS-Base.repo https://mirrors.aliyun.com/repo/Centos-vault-8.5.2111.repo
```

其他发行版 / 版本可参考阿里云镜像站：[https://developer.aliyun.com/mirror/](https://developer.aliyun.com/mirror/)

#### 三、添加 EPEL 源（扩展软件仓库）
EPEL（Extra Packages for Enterprise Linux）提供了官方源中没有的大量开源软件，建议安装。

#### 1. 安装 EPEL 源
```bash
# CentOS 7
sudo yum install epel-release

# 若官方源安装失败，可直接下载阿里云的 EPEL 源
sudo wget -O /etc/yum.repos.d/epel.repo https://mirrors.aliyun.com/repo/epel-7.repo

# CentOS 8
sudo dnf install https://dl.fedoraproject.org/pub/epel/epel-release-latest-8.noarch.rpm
```

#### 四、手动配置自定义 yum 源（高级操作）
若需要添加特定软件的仓库（如 Docker、Nginx 官方源），可手动创建 `.repo` 配置文件。

#### 示例：添加 Docker 官方源
1. 创建 repo 文件：
```bash
sudo vim /etc/yum.repos.d/docker-ce.repo
```

2. 写入以下内容（根据官方文档获取最新地址）：
```plain
[docker-ce-stable]
name=Docker CE Stable - $basearch
baseurl=https://download.docker.com/linux/centos/$releasever/$basearch/stable
enabled=1
gpgcheck=1
gpgkey=https://download.docker.com/linux/centos/gpg
```

    - `[docker-ce-stable]`：仓库唯一标识（不能重复）
    - `name`：仓库名称（自定义）
    - `baseurl`：仓库地址（必须正确）
    - `enabled=1`：启用该仓库（0 为禁用）
    - `gpgcheck=1`：验证软件包签名（安全推荐开启）

#### 五、验证与更新 yum 源
配置完成后，需更新 yum 缓存使配置生效：

#### **清理旧缓存**：
```bash
sudo yum clean all
```

#### **生成新缓存**：
```bash
sudo yum makecache  # 或 sudo dnf makecache
```

#### **验证源是否生效**：
```bash
# 列出所有可用仓库
yum repolist all

# 搜索一个软件（如 nginx），测试是否能从新源获取
yum search nginx
```

#### 六、常见问题解决
1. **“Could not retrieve mirrorlist” 错误**：
    - 检查网络连接（`ping mirrors.aliyun.com`）
    - 确认 repo 文件中的 `baseurl` 是否正确（尤其是版本号）
2. **GPG 验证失败**：
    - 临时禁用 GPG 检查：`sudo yum install 软件名 --nogpgcheck`
    - 或添加正确的 `gpgkey` 地址到 repo 文件
3. **源冲突**：
    - 禁用不需要的仓库：`sudo yum-config-manager --disable 仓库名`
    - 或直接删除对应的 `.repo` 文件

通过配置合适的 yum 源，能显著提升软件安装速度和可用性，国内用户优先推荐阿里云、网易等镜像源。

## 二、Debian 系（Ubuntu、Debian）
主要使用 `apt`（日常简化命令）和 `apt-get`（底层命令），推荐使用 `apt` 更简洁。

| **操作** | `**apt**`<br>**命令** |
| :--- | :--- |
| **更新软件源缓存** | `sudo apt update` |
| **安装软件** | `sudo apt install 软件名` |
| **卸载软件（保留配置）** | `sudo apt remove 软件名` |
| **彻底卸载（删除配置）** | `sudo apt purge 软件名` |
| **升级指定软件** | `sudo apt install --only-upgrade 软件名` |
| **升级所有软件** | `sudo apt upgrade` |
| **搜索软件** | `apt search 关键词` |
| **查看软件信息** | `apt show 软件名` |
| **列出已安装软件** | `apt list --installed` |
| **清理无用依赖** | `sudo apt autoremove` |


**示例**：

```bash
# 更新缓存并安装 MySQL
sudo apt update && sudo apt install mysql-server

# 升级系统所有软件
sudo apt upgrade

# 彻底卸载 LibreOffice
sudo apt purge libreoffice*
```

## 三、通用补充命令（适用于多数发行版）
1. **源码编译安装**（无预编译包时使用）：
```bash
# 下载源码包
wget https://example.com/software.tar.gz

# 解压
tar -zxvf software.tar.gz && cd software

# 配置、编译、安装
./configure --prefix=/usr/local/software  # 自定义安装路径
make
sudo make install
```

2. **二进制包安装**（预编译包，无需编译）：
```bash
# 下载二进制包（以 Node.js 为例）
wget https://nodejs.org/dist/v20.9.0/node-v20.9.0-linux-x64.tar.xz

# 解压并移动到系统目录
tar -Jxvf node-v20.9.0-linux-x64.tar.xz
sudo mv node-v20.9.0-linux-x64 /usr/local/nodejs

# 添加到环境变量（方便直接运行）
echo 'export PATH=/usr/local/nodejs/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

3. **查看软件安装路径**：
```bash
which 软件名    # 查看可执行文件路径（如 which python3）
whereis 软件名  # 查看软件相关文件路径（如 whereis nginx）
```

4. **验证软件是否安装**：
```bash
dpkg -l 软件名  # Debian 系（如 dpkg -l nginx）
rpm -q 软件名   # RedHat 系（如 rpm -q nginx）
```

## 四、注意事项
1. 执行安装 / 卸载 / 升级操作通常需要管理员权限（`sudo` 或 `root`）。
2. 优先使用官方仓库安装软件，安全性和兼容性更有保障。
3. 升级系统前建议备份重要数据，避免版本冲突导致问题。
4. 第三方软件（源码 / 二进制包）需手动管理升级和卸载，建议记录安装路径。

根据使用的 Linux 发行版选择对应命令，熟练掌握包管理工具能高效完成软件全生命周期管理。
